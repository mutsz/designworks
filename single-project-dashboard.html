Why building?

In the current Dnanexus system, users need to browse through the long list of projects (there could be 100-1000 of them) and dig deeper in each in order to exploit work status of jobs.



“The long list of projects currently can be intimidating.” 

-- User A



“It would be very helpful at the projects table to have an icon indicating how many running jobs a project has, and if none, then the status of the most recently finished job, via a traffic light. You could quickly see which projects needed some attention to push past a ‘failed’ analysis, and to see which projects are still running jobs.”

-- User B



“I would love to see running tasks / recently run tasks, as I usually will log in just to check on those..” 

-- User C



(Previous first page of DNAnexus system where users could find it difficult to focus on the essential tasks.)


A dashboard, a Solution,
is needed.

Having a space where allows user to monitor jobs they care, remind them of anything that goes wrong to prevent unwanted money waste and continue on their jobs instantly is the mission of dashboard project. Two goals are defined to achieve.


1. Evaluate overall job status across projects at a glance.

2. Find risks, and feel in control.

In order to achieve these goals, we came up with the next question: What are some features that help users achieve both goals? To answer this question, we traced back to history feedbacks from customers, brainstormed with stakeholders, and interviewed current scientist teams.. we came up with several ideas and conclude 6 main tasks to focus.


Examine billing information.

Search data.

Upload and monitor data.

View jobs of various status.

Access to featured projects.

Notify to remind.

Six Wireframes Iterations

Wireframe 1 - Seperate boxs

Each feature is first designed in separated boxes in a page. Basically, the idea is creating a space where users’ views could jump between different information quickly. Also, this version aims to be a bait for prompting more feedbacks from interviewees.



Wireframe 2 - Optimize interfaces and make space efficiently used.

The space for Upload Data, notification are more efficently used.
The widges become customizable.
